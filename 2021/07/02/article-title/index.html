尝试搭建博客

今天初步使用一下hexo

Git

  1. 登陆 GIT官网 下载安装

  2. 设置用户名与邮箱

  3. 初始化本地库

    命令 功能
    git init 初始化本地库
    git status 查看本地库状态
    git add 文件 添加文件到暂存区
    git rm –cached 文件 删除暂存区内文件
    git commit -m “日志信息” 文件 暂存区提交到本地库
    git reflog 查看日志信息
    git log 查看详细日志信息
    git reset –hard 版本号 版本穿梭
    1. 修改文件后，使用 git status会标红修改后未提交到文件，再使用git add 文件和git commit -m “日志信息” 文件即可提交新版本
    2. 版本穿梭的版本号由git reflog获得，精简和完整的版本号都可以

  4. git分支操作

    命令 功能
    git branch -v 查看分支状况
    git branch 分支名称 创建分支
    git checkout 分支名称 转到分支
    git merge 需要合并下来的分支名称 合并分支

  5. git远程库

    命令 功能
    git remote -v 现实远程库别名
    git remote add 要设置的别名 远程库https地址 设置远程库别名
    git push 远程库别名 要推送的分支名 推送到远程库
    git pull 远程库别名 要拉取的分支名 拉去远程库
    git clone 远程库地址 克隆库
    1. 注：克隆库的别名为origin
    2. 邀请管理者方法项目中Setting-Manage access-Invite a collaborator邀请用户
    3. 跨团队协作：点击项目右上角的fork，即放到自己的远程库，可以修改后提交

  6. ssh密钥

    1. 终端输入ssh-keygen -t rsa -C github登陆邮箱
    2. 连按回车直到成功
    3. cd /Users/yangzhuoran/.ssh跳转到文件目录下，id_rsa为私钥，id_rsa.pub为公钥
    4. cat id_rsa.pub打开公钥，复制到github-头像-setting-SSH and GPG keys-New SSH key生成公钥

  1. git常用命令

    命令 功能
    git add 增加文件到暂存库
    git commit
    git pull 先拉，再提交
    git push
